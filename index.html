ðŸ”— Blockchain Layer (EigenLayer AVSs & Ethereum L2)

Canvas Protocol operates as a Verifiable Application, integrating AVS technology on EigenLayer, with its B2B marketplace on an Ethereum L2. This provides:

  • Decentralized Trust & Security via EigenLayer Restaking:
    • AVS operators restake ETH, providing high economic security for validating interactions.
    • Attributable fault with slashing for misbehavior ensures operator honesty and the trustworthiness of attestations.
    • Shared security from Ethereum reduces risks compared to isolated systems.
  • Core Marketplace Operations on Ethereum L2:
    • Handles efficient B2B payments, ad auctions, and order matching for the marketplace.
  • Integrated AVS Services for End-to-End Verifiability:
    • Decentralized Identity/Personhood AVS: Provides "Proof of Human Interaction" by validating CAPTCHA telemetry and user uniqueness (e.g., via privacy-preserving behavioral indicators or future anonymous proof tokens), crucial for verifying genuine engagement.
    • Data Validation AVS: For data collection CAPTCHAs (AI training, surveys), AVS operators verify data quality (e.g., against ground truth or via consensus) and process compliance, generating on-chain attestations of data integrity for clients.
    • Oracle AVSs: Securely bring verified interaction counts, campaign metrics, and settlement data from the L2 marketplace on-chain (or make it accessible to other on-chain contracts) for enhanced transparency and advertiser reporting.
    • (Future) Incentive Mechanism AVS: To manage and distribute Phase 2 user rewards based on AVS-verified interactions. (Operators and restakers earn a portion of protocol fees for these AVS services).
  • On-Chain Accountability: Irrevocable, transparent audit trail of interactions and attestations, building trust with all platform participants.
  • Interaction Model: Canvas Protocol (on L2) makes on-chain calls to AVS contracts for critical proofs/attestations and may use off-chain communication with AVS operator nodes for processing detailed CAPTCHA telemetry before succinct on-chain validation.

ðŸš€ Future Vision: Phase 2 - User Rewards & Ecosystem Growth

Once the B2B marketplace is established (Phase 1), Canvas Protocol will introduce direct crypto rewards (e.g., ETH/Stablecoins) to end-users for successfully completing interactive CAPTCHAs. This will create a three-sided economy, further incentivizing user participation, increasing the value of the CAPTCHA inventory for publishers, and providing an even more engaged audience for requesters. This phase aims to drive broader adoption of the Ethereum L2 and EigenLayer ecosystem and establish a new standard for user verification online. Further scaling could involve deploying Canvas Protocol on its own dedicated, high-performance rollup, utilizing EigenDA for efficient and secure data availability to manage the potentially vast scale of CAPTCHA interactions and marketplace data.
